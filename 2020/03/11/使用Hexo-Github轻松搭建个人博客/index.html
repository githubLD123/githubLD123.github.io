使用Hexo+Github轻松搭建个人博客

发表于 2020-03-11 更新于 2020-03-12 分类于 个人博客搭建

github给每个账户提供了一个域名，这给我们搭建个人站点提供了方便。使用github pages服务搭建博客的好处有：

  1. 全是静态文件，访问速度快；
  2. 免费方便，不用花一分钱就可以搭建一个自由的个人博客，不需要服务器不需要后台，主要是没有广告呀！
  3. 可以随意绑定自己的域名，不仔细看的话根本看不出来你的网站是基于github的；
  4. 数据绝对安全，基于github的版本管理，想恢复到哪个历史版本都行；
  5. 博客内容可以轻松打包、转移、发布到其它平台；

环境配置

安装Node.js

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。
Node.js传送门

NPM 使用介绍

NPM是随同NodeJS一起安装的包管理工具，能解决NodeJS代码部署上的很多问题，常见的使用场景有以下几种：

  1. 允许用户从NPM服务器下载别人编写的第三方包到本地使用。
  2. 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
  3. 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。
    由于新版的nodejs已经集成了npm，所以安装Node.js的时候npm也一并安装好了。可以通过输入 来测试是否成功安装，出现版本提示表示安装成功。

安装Git & 配置Github

[Github传送门]
GitHub是一个面向开源及私有软件项目的托管平台，因为只支持git 作为唯一的版本库格式进行托管，故名GitHub。是当前最流行的技术开源社区，当然你也可以把它当作论坛，云盘，托管项目，管理任务等等。

安装Git

Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。关于Git的使用，你可以参照Git 完整命令手册或者点击Git简明指南查看，资料来自菜鸟教程。

配置Github

注册Github账户，怎么注册账户就不说了…

创建仓库

新建一个名为你的用户名.github.io的仓库，比如说，如果你的github用户名是test，那么你就新建test.github.io的仓库（必须是你的用户名，其它名称无效），将来你的网站访问地址就是 http://test.github.io 了。

注意：

  1. 注册的邮箱一定要验证，否则不会成功；
  2. 仓库名字必须是：username.github.io，其中username是你的用户名；

绑定域名

如果你有自己的域名那就绑定一下，没有也不影响。
点击新建的仓库标签找到setting->option->github pages->custom domain输入域名，保存，完事儿。（注意自己的域名要配有云解析否则不能用）

Hexo

Hexo是一个简单、快速、强大的基于 Github Pages 的博客发布工具，支持Markdown格式，有众多优秀插件和主题。
Hexo传送门
Hexo文档

安装之前先来说几个注意事项：

  1. 很多命令既可以用电脑本来的终端来完成，也可以使用git bash来完成，但是部分命令会有一些问题，为避免不必要的问题，建议全部使用git bash来执行；
  2. hexo不同版本差别比较大，网上很多文章的配置信息都是基于2.x的，所以注意不要被误导；
  3. hexo有2种配置文件_config.yml文件，一个是根目录下的全局的_config.yml（管站点的配置），一个是各个主题的配置文件，管主题的配置。
